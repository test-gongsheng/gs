# 2026-04-01

## 股票监控系统修复

### 问题：持仓列表不显示实时数据
**根因**：`index.html` 底部定义的 `renderStockList` 函数覆盖了 `app.js` 中的同名函数，导致持仓列表渲染失败。

**修复内容**：
1. 在 `app.js` 中添加 `_updatingQuotes` 锁，防止 `updateStockPricesOnce` 和 `simulatePriceUpdate` 竞争条件
2. 将 `index.html` 中的函数重命名为 `renderPortfolioStockList`，避免命名冲突
3. 修改保底渲染逻辑，从替换数据改为合并数据，保留已有行情数据
4. 移除 `renderStockList` 中的强制内联样式（特别是绿色边框），恢复 CSS 类定义的样式

**验证结果**：持仓列表正确显示 12 只股票，价格和涨跌幅实时更新，样式与整体风格一致。

## 教训与反思

### 测试纪律
用户提醒：「不是说每次修改都要完整测试吗？」
- 不能仅看日志输出正常就认为修复成功
- 必须实际查看 UI 渲染结果
- 命名冲突这类问题在代码审查时容易忽略，需要通过实际运行发现

### 代码组织
- 同一项目中避免同名函数，即使是不同模块
- HTML 中嵌入的 JS 代码可能覆盖外部文件定义的函数
- 使用描述性更强的函数名可以减少冲突风险

## 技术细节备忘

**竞争条件解决方案**：
```javascript
// appState 中添加锁
_updatingQuotes: false

// 在行情更新函数中
if (appState._updatingQuotes) return;
appState._updatingQuotes = true;
try {
    // 更新逻辑
} finally {
    appState._updatingQuotes = false;
}
```

**Flask 模板缓存**：修改 templates/index.html 后需要重启服务才能生效。
