# 2026-04-02 工作记录

## 南向资金数据问题修复

### 问题现象
用户反馈持仓股中3只港股（腾讯00700、阿里巴巴09988、比亚迪电子00285）的南向资金数据显示一样，且快速切换时出现数据错乱或无数据。

### 修复过程

#### 第一轮修复 (commit 2e8510b)
- **问题**: 前端 reduce 累加时 net_inflow 可能是字符串，导致拼接而非数值相加
- **修复**: `d.net_inflow` → `parseFloat(d.net_inflow) || 0`
- **结果**: 计算正确，但仍有竞态条件问题

#### 第二轮修复 (commit d5fa1c6)
- **问题**: 快速切换股票时异步请求返回顺序错乱
- **修复**: 添加 AbortController 和 currentSouthboundStockCode 验证
- **结果**: 请求取消机制有问题，导致数据完全加载失败

#### 第三轮修复 (commit bed96fc)
- **问题**: AbortController 导致请求被意外取消
- **修复**: 移除 AbortController，简化竞态条件处理
- **结果**: 仍有数据错乱问题

#### 第四轮修复 (commit 677e85d)
- **问题**: 港股代码匹配逻辑不完善
- **修复**: 改进代码标准化匹配逻辑，增加调试日志
- **结果**: 日志显示请求堆积严重

#### 第五轮修复 (commit f709a7f)
- **问题**: requestId 机制导致所有响应被丢弃
- **修复**: 重写 requestId 竞态条件防护
- **结果**: 响应被过度丢弃

#### 第六轮修复 (commit b560144)
- **问题**: app.js 每次价格更新都调用 loadSouthboundStockData，导致请求堆积
- **修复**: 
  1. app.js 只在股票代码变化时才调用 loadSouthboundStockData
  2. southbound.js 简化竞态条件防护，只保留基本检查
- **状态**: 待验证

### 关键代码变更

**app.js 加载逻辑:**
```javascript
// 只在股票代码变化时才加载南向资金数据
if (typeof loadSouthboundStockData === 'function' && 
    appState.lastSouthboundStockCode !== stock.code) {
    appState.lastSouthboundStockCode = stock.code;
    loadSouthboundStockData(stock.code);
}
```

**southbound.js 简化防护:**
```javascript
async function loadSouthboundStockData(stockCode) {
    currentSouthboundStockCode = stockCode;
    // ... fetch ...
    // 检查是否仍然是当前选中的股票
    if (currentSouthboundStockCode !== stockCode) {
        return;  // 已切换，忽略
    }
}
```
