## 中轴价格Bug修复

### 问题发现
用户反馈中轴价格显示错误（显示为持仓成本387.34，而非真实中轴价格）

### 根因分析
1. 导入时把 `costPrice` 当成了 `pivotPrice`
2. `refreshAxisPrices()` 只更新前端和localStorage，没同步后端数据库

### 修复方案
1. `import.js` - 导入时不再预设中轴价格（设为0）
2. `app.js` - 刷新中轴价格后调用 `/api/stocks/{id}/axis` 同步保存到后端

- commit 97e56ed: 升级数据源为akshare

## 港股沽空数据接入

### 背景
用户选择"方案A"，要求接入港交所官方披露数据，实现真实数据源。

### 尝试过的方案
1. **港交所官网** - 页面404，已迁移
2. **披露易网站** - 获取到的是港股通持股数据，非沽空数据
3. **东方财富/同花顺/雪球** - 需授权或接口失效
4. **akshare** - 无专门港股沽空接口，网络不稳定

### 最终方案
由于港交所沽空数据T+1披露机制，无法获取实时数据。改为：
- 不再返回估算数据（原12.5%为估算）
- 前端显示"港交所T+1披露，数据待更新"
- 提示用户通过富途/雪球查看实时估算

### 代码改动
- `utils/market_sentiment.py` - 移除估算逻辑，返回`data_pending`标记
- `static/js/app.js` - 根据`data_pending`显示相应UI

### 修复

Uncaught SyntaxError: Unexpected token 'catch'
