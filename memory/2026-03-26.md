# 2026-03-26 股票监控系统修复记录

## 上午会话 - 持仓导入与数据显示问题

### 问题1: 导入股票数量不对
- 症状: 后端只有1只股票，前端显示12只，重复数据
- 原因: localStorage缓存与后端数据不同步
- 修复: 增加版本号强制清理缓存 (2.5.1 → 2.6.2)

### 问题2: 批量导入并发冲突
- 症状: 导入多只时只有部分保存成功
- 原因: 多个POST请求同时到达，数据文件读写冲突
- 修复: 新增 `/api/stocks/batch` 批量导入接口，一次性写入所有股票

### 问题3: 股票ID冲突
- 症状: 多个股票获得相同ID
- 原因: `max_id + 1` 在并发时重复
- 修复: 改用 `时间戳 + 3位随机数` 生成唯一ID

### 问题4: 港股持仓市值计算错误
- 症状: 系统显示636万，交易软件显示627万
- 原因: 前端计算时港股价格未正确转换为人民币
- 修复: 使用 `price_cny` 字段或 `/ exchangeRate` 转换

### 问题5: 404错误持续出现
- 症状: 刷新中轴价格时大量404
- 原因: 前端ID与后端不匹配
- 修复: 自动创建机制，遇到404时POST创建股票

### 问题6: 港股沽空趋势不显示
- 症状: 个股沽空建议显示"--"
- 原因: HTML元素ID是 `hkIndividualAdvice`，JS查找 `hkIndividualRiskAdvice`
- 修复: 统一ID名称

### 问题7: 当日盈亏计算错误
- 症状: 盈亏数值明显不对
- 原因: 公式错误 `(市值-成本)*涨跌幅/100`
- 修复: 改为 `持仓数量 × 涨跌额`（港股需汇率转换）

## 技术决策

### 汇率策略
- A股: 直接使用人民币价格
- 港股: 使用昨日收盘汇率（与交易软件一致）
- API返回: `price` (港币) 和 `price_cny` (人民币)

### 缓存策略
- 版本号触发: `APP_VERSION` 变更时强制清理 localStorage
- 数据优先: 后端API优先，失败时fallback到localStorage

### 并发处理
- 批量接口: `/api/stocks/batch` 一次性添加所有股票
- 清空接口: `/api/stocks/clear` 导入前先清空旧数据

## 下午会话 - 港股市值计算修复完成

### 问题4-1: 港股市值计算根本原因
- 症状: 后端返回599.71万，系统显示635.8万，差36万
- 根因: 行情API返回港股 `price` 是港币，`price_cny` 是人民币；`updateAssetOverview` 用 `price` 除以汇率计算，但汇率可能在不同地方被覆盖
- 修复: 直接使用 `priceCny` 字段计算港股市值，避免汇率转换错误
- 代码: `const cnyPrice = stock.priceCny || ((stock.price || 0) / exchangeRate);`
