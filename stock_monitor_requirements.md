# 股票投资监控系统需求文档

## 一、系统概述

**总投资额度**: 600万元  
**投资分布**:
- A股投资上限：50万元/只（个别可达100万元）
- 港股投资上限：150万元/只

**核心策略**: 中轴价格仓位控制法

---

## 二、监控界面布局

### 2.1 自选股票显示区

#### 功能要求
- 支持手动实时输入新的自选股票
- 支持修改已有股票的持仓信息

#### 股票信息字段
| 字段 | 说明 |
|------|------|
| 股票代码 | 股票唯一标识 |
| 股票名称 | 显示名称 |
| 市场类型 | A股/港股 |
| 投资上限 | 该股票最大投入金额 |
| 当前持仓数量 | 已购买股数 |
| 当前持仓成本 | 平均持仓成本 |
| 当前市值 | 实时计算 |
| 中轴价格 | 策略核心参考价（由系统综合评估给出） |
| 底仓比例 | 固定持仓比例（如50%） |
| 浮动仓比例 | 可调整仓位比例（如50%） |
| 当前仓位状态 | 满仓/减仓/增仓提示 |
| 下次操作触发价 | 上涨8%或下跌8%的目标价 |

#### 操作策略（中轴价格仓位控制法）
```
基准：中轴价格（由AI综合基本面、技术面、市场情绪动态计算）

仓位结构：
- 底仓：50%（长期持有，不轻易变动）
- 浮动仓：50%（根据价格波动调整）

浮动仓调整规则：
- 当股价较中轴价格上涨8% → 减持浮动仓的20%
- 当股价较中轴价格下跌8% → 增持浮动仓的20%

示例：
- 中轴价格：100元
- 投资上限：100万
- 底仓：50万（5000股）
- 浮动仓：50万

上涨场景：
- 股价涨至108元（+8%）→ 减持浮动仓20%（即10万市值）
- 实际卖出：约925股

下跌场景：
- 股价跌至92元（-8%）→ 增持浮动仓20%（即10万现金买入）
- 实际买入：约1087股
```

## 三、实时数据要求

### 3.1 数据更新频率
| 数据类型 | 更新频率 | 备注 |
|---------|---------|------|
| 股价数据 | 实时（秒级） | 交易时间内 |
| 持仓盈亏 | 实时计算 | 基于最新股价 |
| 新闻推送 | 实时 | 重要新闻立即推送 |
| 市场情绪评估 | 每30分钟 | 交易时间内 |
| 热点板块扫描 | 每小时 | 交易时间内 |
| 日终报告 | 收盘后 | 每日总结 |

### 3.2 重点提醒机制

#### 买卖提醒（最高优先级）
- **触发条件**: 股价达到预设的±8%操作线
- **提醒方式**: 即时弹窗 + 声音提示 + 推送通知
- **提醒内容**: 
  - 股票名称/代码
  - 当前价格
  - 涨跌幅
  - 建议操作（增持/减持）
  - 建议数量/金额
  - 触发原因（到达中轴±8%位置）

#### 策略触发提醒
- **触发条件**: 
  - 中轴价格需要调整（基本面重大变化）
  - 投资上限需要调整（资金分配变化）
  - 底仓/浮动仓比例需要调整（策略优化）
- **提醒方式**: 弹窗提示 + 详细说明

#### 新闻热点推送
- **推送类型**:
  - 重大利好/利空（立即推送）
  - 行业政策变化（立即推送）
  - 公司公告（立即推送）
  - 机构评级调整（立即推送）
  - 市场热点综述（每小时汇总）

## 六、技术实现建议

### 6.1 数据源
- 股票行情：新浪财经/腾讯财经API
- 新闻资讯：财联社API、东方财富API
- 研报数据：Choice数据/Wind数据（如有权限）

### 6.2 提醒方式
- 桌面端：弹窗 + 声音
- 移动端：APP推送（如支持）
- 备用：邮件/短信提醒（重要信号）

### 6.3 部署方式
- 本地运行：Python + Web界面（Flask/Streamlit）
- 定时任务：cron job 定时抓取数据
- 实时监控：WebSocket 推送价格变化

## 七、使用流程

### 7.1 初始化设置
1. 添加自选股，输入基本信息
2. 设置投资上限
3. 系统计算初始中轴价格
4. 确认底仓/浮动仓比例
5. 开启监控提醒

### 7.2 日常操作
1. 开盘前：查看隔夜新闻、国际市场动态
2. 交易中：接收实时提醒、查看操作建议
3. 触发买卖线：根据提醒执行交易（或确认）
4. 收盘后：查看日终报告、调整策略参数

### 7.3 定期维护
- 每周：回顾中轴价格合理性，必要时调整
- 每月：评估策略执行效果，优化参数
- 每季：重新审视投资上限分配

## 八、风险控制

### 8.1 单只股票风险
- 严格执行投资上限
- 底仓保护机制（不轻易清仓）
- 止损线设置（如-20%强制止损）

### 8.2 整体仓位风险
- 总仓位监控（不超过600万）
- 行业分散度检查
- 市场系统性风险预警

### 8.3 操作风险
- 买卖确认机制（防止误操作）
- 交易时间限制（非交易时间不提醒）
- 价格异常过滤（防止数据错误）
